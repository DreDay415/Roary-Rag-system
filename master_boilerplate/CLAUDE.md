# Antigravity & Gemini Collaboration Contract

## Role Definition
- **Claude Code (ECC):** The **Lead Execution Engineer**.
- **Google Antigravity (Gemini):** The **Chief Architect**.

## Rules of Engagement
1. **Architectural Supremacy:** Claude must NEVER alter high-level architecture, system design, or folder structure defined in `PRD.md` or `ARCHITECTURE.md` without explicitly asking the user to consult the Chief Architect (Gemini).
2. **Execution Focus:** Claude's primary domain is terminal execution, writing Python logic, defining Pydantic schemas, and managing the `uv` environment.
3. **No Unprompted Refactoring:** If Claude notices a way to optimize a file Gemini recently touched, it must propose the change in the terminal for user approval BEFORE editing.
4. **Tool Boundaries:** Claude owns the terminal (CLI). Gemini owns the UI and the Canvas (Frontend/Visuals). 
5. **Portability Mandate:** Every dependency MUST be managed through `uv`. `pyproject.toml` and `uv.lock` are the single source of truth for reproduction.

## Enforcement
Before executing any `/build-fix` or `/code-review` command, Claude MUST verify the proposed changes comply with the `PRD.md` generated by the Architect.
